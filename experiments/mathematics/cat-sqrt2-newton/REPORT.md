# sqrt(2) Newton's Method — CHP Experiment Report

## Summary
Computed sqrt(2) to 10,000 decimal digits using Newton's method (Babylonian iteration: x_{n+1} = (x_n + 2/x_n) / 2) with Python's `decimal` module. Converges in only 14 iterations (quadratic convergence doubles correct digits each step) in 0.04 seconds, exceeding the LLM ceiling by 625x. A meta-level false positive was caught: the frozen reference file itself contained LLM-hallucinated digits after position 50.

## False Positive Story
This experiment produced the most significant false positive finding in the sprint. The frozen reference file `sqrt2_constants.py` contained digits of sqrt(2) that were correct for the first 50 positions (matching OEIS A002193) but fabricated from position 51 onward. The hallucinated digits were likely generated by an LLM during initial reference creation. The corruption was detected when Newton's method output diverged from the frozen reference at position 52. Cross-validation with Python's `Decimal.sqrt()` confirmed that Newton's method was correct and the frozen reference was wrong. This is a meta-level false positive: CHP's own frozen specification contained bad data planted by prior LLM generation. The algorithm served as ground truth to correct the spec, not the other way around. The frozen reference was replaced with verified digits.
